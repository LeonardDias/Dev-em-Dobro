Síncrono vs Assíncrono

Execuções síncronas no JavaScript ocorrem de forma sequencial, linha por linha. Cada linha de código deve ser concluída antes de passar para a próxima. Esse tipo de execução é simples e previsível, mas pode ser problemático em operações demoradas, pois o código espera até que a operação atual termine antes de continuar.
O JavaScript assíncrono permite que operações possam ser executadas em segundo plano enquanto o resto do código continua a ser executado. Isso evita que o código “trave” em operações demoradas e melhora a experiência do usuário, especialmente em aplicações de longa execução e carregamento de dados externos (como APIs).
